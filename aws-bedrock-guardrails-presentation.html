Why AWS was needed

+

No cloud access
FastAPI ran on localhost. Portfolio data was unreachable from any other device or network. The app only worked on one machine.

SecureString parameters are encrypted at rest using AWS KMS. If the server is compromised, no credentials are exposed from disk. INTERNAL_API_KEY authenticates the Lambda → FastAPI internal endpoint. CORS_ORIGINS controls which domains can call the API.
# backend/main.py — runs before any routes load
 
def _load_ssm_secrets():
  ssm = boto3.client(
    "ssm",
    region_name=os.getenv("BEDROCK_REGION")
  )
  kwargs = {
    "Path": "/investment-manager/",
    "WithDecryption": True
  }
  while True:
    resp = ssm.get_parameters_by_path(**kwargs)
    for p in resp["Parameters"]:
      key = p["Name"].split("/")[-1]
      os.environ[key] = p["Value"]
    if "NextToken" not in resp: break
    kwargs["NextToken"] = resp["NextToken"]
 
_load_ssm_secrets() # called at module load

1 million free invocations/month forever

10 — SUMMARY

Takeaways

+
WHAT I LEARNED
SSM Parameter Store is the right place for secrets — not .env files on disk
IAM Roles eliminate hardcoded credentials entirely — no keys anywhere
CloudWatch + watchtower gives full log visibility without SSH
Bedrock Guardrails blocks at infrastructure level — zero application code needed
Lambda + EventBridge replaces cron daemons with serverless scheduling
WHAT WAS DIFFICULT
SSM pagination — default limit is 10 params, needed NextToken loop
IAM policy ARN format for Bedrock models and guardrails
guardrailIdentifier vs guardrailVersion API parameter naming
Handling guardrail_intervened gracefully in the frontend
Nginx 300s timeout config for long AI chat responses
IMPROVE IN PRODUCTION
Move to ECS/Fargate — no instance management, true autoscaling
CloudWatch alarm when guardrail block rate exceeds 5%
SES production access — currently sandbox (verified addresses only)
RDS or DynamoDB instead of Atlas for fully-integrated AWS stack
WAF in front of ALB for HTTP-layer protection
