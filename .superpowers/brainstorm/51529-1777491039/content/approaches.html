3 implementation approaches

All produce the same Option C pill result — they differ in how the transition is handled.

A

Instant swap — no animation

A toggle button in each section header. Click → section disappears instantly, a small pill button appears in its place. Click pill → section reappears. Pure JS class toggle + CSS display:none. Zero animation complexity.

Pros

  • Simplest code
  • No layout edge cases
  • Fastest to implement

Cons

  • Abrupt, no visual transition
B ★ recommended

CSS max-height animation

Toggle button in header. Click → content area animates to max-height:0; overflow:hidden in ~200ms, then pill fades in. Clean, works well with the existing flexbox layout. A known quirk: the ease-out on expand can feel slightly slow if the content is tall (200px chart, chat messages).

Pros

  • Smooth visual transition
  • CSS-only animation (no JS height math)
  • Fits existing layout perfectly

Cons

  • Transition duration feels slightly uneven at different heights
C

JS measured height animation

JS measures the actual rendered height of the content, sets it explicitly, then transitions to height:0. Most precise animation — easing looks identical on collapse and expand. Requires a small JS helper to compute heights before each transition.

Pros

  • Perfectly smooth and symmetric animation

Cons

  • More JS code
  • Must recalculate if content changes size (chart resize, new messages)
